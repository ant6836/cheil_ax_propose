# 제일기획 AI 전략 인텔리전스 플랫폼 — POC 대시보드 개발 요구서

> **목적**: 제일기획에게 제안하는 AI 기반 전략 인텔리전스 플랫폼의 예상 결과물을 시각적으로 시연하기 위한 POC 웹 대시보드 개발.  
> 실제 서비스 배포가 아닌 **데모 시연용**이며, 모든 데이터는 Mock 데이터로 구성한다.

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 제일기획 AI 전략 인텔리전스 플랫폼 POC |
| 개발 형태 | 단일 페이지 웹 애플리케이션 (SPA) |
| 데이터 | 전체 Mock 데이터 (가상 데이터) |
| 목적 | 클라이언트 제안 시연용 POC |

---

## 2. 기술 스택

- **프레임워크**: React (JSX)
- **차트 라이브러리**: Chart.js (scatter, bubble chart)
- **그래프 시각화**: D3.js 또는 React Force Graph (온톨로지 탭)
- **스타일**: Tailwind CSS (core utility only) 또는 inline CSS
- **상태 관리**: React useState / useEffect

### 3.1 헤더 공통 요소

- **로고**: 제일기획 CI (텍스트 또는 이미지)
- **타이틀**: "AI 전략 인텔리전스 플랫폼"
- **마지막 업데이트**: `YYYY-MM-DD HH:mm 기준` 형식, 모든 탭 공통 노출
- **새로고침 버튼**: 클릭 시 "분석 중..." 상태 표시 → 완료 시 타임스탬프 갱신

  - 로딩 중 레이블: `분석 중...`
- **업데이트 주기**: 일 단위 자동 업데이트 (데모에서는 버튼으로 수동 트리거)

## 4. 탭 구성 (6개)

| 순서 | 탭명 | 설명 |
|------|------|------|

## 5. 공통 컴포넌트

### 5.1 지역 선택 버튼

Weak Signal, Opportunity Map, Ontology 탭 상단에 공통 적용.

```
[ 국내 ] [ 북미 ] [ 유럽 ]
```

- 선택된 지역은 버튼 강조 (검정 배경 + 흰 텍스트)
- 지역 전환 시 차트와 AI 해석 패널이 동시 교체
- 추후 확장 예정 지역: 중동, 동남아, 중남미 (데모에서는 비활성 표시)

### 5.2 AI 해석 패널

차트 오른쪽 또는 하단에 고정 배치. 모든 분석 탭에 적용.

구성 요소 (3개 항목):
1. **감지된 주요 신호 / 핵심 기회** — 현재 무슨 신호·기회가 감지됐는가
2. **사업 영향** — 제일기획의 어떤 사업 영역에 어떻게 영향을 주는가
3. **권고 액션** — 제일기획이 취해야 할 다음 행동은 무엇인가

> POC에서는 하드코딩된 Mock 텍스트로 구성. 탭·지역 전환 시 해당 데이터에 맞는 Mock 해석 텍스트로 교체.

### 5.3 북마크 & 메모

- 모든 탭의 개별 항목(신호, 기회, 노드 등)에 북마크 아이콘(☆) 표시
- 클릭 시 ★로 전환, 메모 입력 팝업 표시
- 북마크된 항목은 Overview 탭의 "저장된 인사이트" 섹션에 집약

## 6. 탭별 상세 요구사항

### 6.1 Overview 탭

**목적**: 경영진이 30초 안에 전체 현황을 파악하는 홈 화면

**레이아웃**:
```
┌──────────┬──────────┬──────────┬──────────┐
│ SWOT 요약 카드 │ Weak Signal TOP 3 │ Opportunity TOP 3 │ 저장된 인사이트 │
└──────────┴──────────┴──────────┴──────────┘
│              최근 업데이트 타임라인              │
```

**구성 요소**:

1. **SWOT 요약 카드** (4분면)
   - S·W·O·T 각 1~2줄 핵심 요약
   - 클릭 시 SWOT 탭으로 이동

2. **Weak Signal TOP 3**
   - 현재 가장 증가율이 높은 신호 3개 카드
   - 각 카드: 키워드명 + 지역 배지 + 증가율 수치 + 신호 강도 게이지
   - 클릭 시 Weak Signal 탭으로 이동

3. **Opportunity TOP 3**
   - 기회점수 상위 3개 카드
   - 각 카드: 기회명 + 지역 배지 + 기회점수 + 중요도/만족도 수치
   - 클릭 시 Opportunity Map 탭으로 이동

4. **저장된 인사이트**
   - 북마크된 항목 리스트
   - 항목명 + 출처 탭 + 날짜

5. **최근 업데이트 타임라인**
   - 최근 7일간 주요 신호 변화 이력
   - 날짜 + 이벤트 설명

### 6.2 SWOT 분석 탭

**목적**: AI가 자동으로 분석·업데이트하는 제일기획 SWOT 및 전략 매트릭스

**데이터 원천**:
- DART 전자공시 사업보고서 (Mock 데이터)
- 제일기획 관련 국내외 최신 뉴스 (Mock 데이터)
- 제일기획 사내 문서 (Mock 데이터)

**레이아웃**:
```
┌────────────────────────────┬──────────────────────┐
│   S·W·O·T 4분면 카드        │   AI 해석 패널         │
│                            │  (SO/ST/WO/WT 전략)   │
├────────────────────────────┴──────────────────────┤
│        전략 매트릭스 (SO / ST / WO / WT)             │
└───────────────────────────────────────────────────┘
```

**SWOT 4분면 카드**:
- S(강점) / W(약점) / O(기회) / T(위협) 각 박스
- 각 항목은 불릿 리스트로 표시
- 각 항목 우측에 북마크 아이콘
- 항목 클릭 시 → 사이드 패널에 해당 항목의 **근거 데이터** 표시
  - 출처 뉴스 기사 제목 + 날짜 + 링크
  - 관련 사업보고서 항목 (Mock)

**전략 매트릭스**:
- SO / ST / WO / WT 4개 전략 카드
- 각 카드: 전략 제목 + AI 해석 텍스트 (2~3줄)
- 분기/반기 단위 변화 비교 토글 버튼

**업데이트 주기 표시**: "마지막 분석: YYYY-MM-DD"

### 6.3 Weak Signal 탭

**목적**: 뉴스 데이터에서 아직 업계 담론이 되지 않은 변화의 초기 신호 감지

**탭 상단 설명 (고정 텍스트)**:
> Weak Signal은 아직 업계 담론이 되지 않은 변화의 초기 신호입니다. Strong Signal이 됐을 때는 이미 경쟁사도 알고 있습니다. 이 탭은 그 이전 단계를 먼저 감지하여 제일기획이 선제적으로 전략 포지션을 잡을 수 있도록 지원합니다.

**데이터 원천**:
- 국내·북미·유럽 웹 뉴스 데이터 (Mock 데이터)
- 광고·마케팅·기술·문화·산업 전반 포괄 (특정 기업·업계 제한 없음)

**레이아웃**:
```
[상단 설명]
[국내] [북미] [유럽]
┌─────────────────────────────┬──────────────┐
│     Weak Signal 스캐터 플롯   │  AI 해석 패널  │
├─────────────────────────────┴──────────────┤
│             키워드 범례                       │
└─────────────────────────────────────────────┘
```

**차트 명세 (Weak Signal 스캐터 플롯)**:

- **차트 타입**: Scatter Plot (Chart.js)
- **X축**: 평균 등장 빈도 (낮을수록 덜 알려짐)
- **Y축**: 시간가중 증가율 (높을수록 최근 급부상)
- **점선**: X축 평균값(수직선) + Y축 평균값(수평선) 각각 표시
- **키워드 분류 (점 색상)**:
  - `[e]` 환경·규제 키워드: 진한 색
  - `[n]` 비즈니스 니즈 키워드: 중간 색
  - `[t]` 기술·제품 키워드: 연한 색
- **Weak Signal 키워드** (좌상단 구간): 더 크고 진한 점, 레이블 표시
- **기타 신호**: 작고 연한 점, 레이블 없음
- **툴팁**: 키워드명 | 빈도: N | 증가율: N

**드릴다운 인터랙션 (키워드 클릭 시)**:
- 사이드 패널 또는 모달 표시
- 해당 키워드를 포함한 뉴스 기사 리스트 (제목 + 날짜 + 출처)
- 시간 흐름에 따른 등장 빈도 추이 선 그래프
- 관련 SWOT 항목 연결 태그 (예: "→ O3 신흥 수주 기회")
- 관련 Opportunity 항목 연결 태그

**신호 강도 게이지**:
- 각 Weak Signal 키워드 카드 하단에 0~100 게이지 바
- 지난 주 대비 변화량 표시 (▲▼ 수치)

**트렌드 타임라인**:
- 특정 키워드 클릭 → 드릴다운 패널에 시계열 선 그래프 포함
- "이 신호는 N개월 전 처음 감지됨" 텍스트

### 6.4 Opportunity Map 탭

**목적**: 소셜미디어 데이터에서 중요도는 높으나 충족도가 낮은 사업 기회 식별

**탭 상단 설명 (고정 텍스트)**:
> 소셜미디어 데이터에서 소비자와 시장의 실시간 반응을 읽어 실행 가능한 사업 기회를 시각화합니다. Weak Signal이 "무엇이 변하고 있는가"를 감지한다면, Opportunity Map은 "그 변화에서 제일기획이 포착할 수 있는 기회는 무엇인가"를 보여줍니다. 특히 **중요도는 높으나 현재 충족도가 낮은 영역**을 우선 식별합니다. 시장이 원하지만 아직 누구도 제대로 채우지 못한 공백이 제일기획의 진입 지점입니다.

**데이터 원천**:
- 국내·북미·유럽 소셜미디어 데이터 (X, Reddit, LinkedIn, TikTok 등) — Mock 데이터
- 특정 기업·브랜드에 국한하지 않은 전반적 트렌드

**레이아웃**:
```
[상단 설명]
[국내] [북미] [유럽]
┌─────────────────────────────┬──────────────┐
│     Opportunity 버블 차트     │  AI 해석 패널  │
├─────────────────────────────┴──────────────┤
│             범례                             │
└─────────────────────────────────────────────┘
```

**차트 명세 (Opportunity 버블 차트)**:

- **차트 타입**: Bubble Chart (Chart.js)
- **X축**: 중요도 (Importance, 0~10)
- **Y축**: 만족도 (Satisfaction, 0~10)
- **버블 크기**: 기회 점수 = `중요도 + max(중요도 - 만족도, 0)` (논문 수식 기반)
- **두 점선**: 우측 상단 (10, 10) 동일 지점에서 출발
  - 점선 1: (10, 10) → X축 평균 중요도값 방향으로 뻗음
  - 점선 2: (10, 10) → Y축 평균 만족도값 방향으로 뻗음
  - 두 점선이 만드는 세 영역: **Underserved** (하단) / **Served Right** (중간) / **Over-served** (상단)
- **Underserved 버블**: 주황색 강조, 레이블 표시
- **기타 버블**: 회색, 레이블 없음
- **툴팁**: 기회명 | 중요도: N | 만족도: N | 기회점수: N

**드릴다운 인터랙션 (버블 클릭 시)**:
- 해당 기회를 구성하는 소셜 포스트 샘플 3~5개 (Mock)
- 감성 분석 수치 (긍정/부정/중립 비율 도넛 차트)
- 제일기획의 연결 가능 강점·사업 영역 태그
- 관련 Weak Signal 항목 연결 태그
- 시간 흐름에 따른 기회점수 추이 선 그래프

### 6.5 Ontology 탭

**목적**: Module 1~3의 분석 결과를 연결하는 전략 지식 그래프 시각화 (Module 4 권고사항 프리뷰)

**레이아웃**:
```
[국내] [북미] [유럽] [전체]
┌───────────────────────────────────┬──────────────┐
│     인터랙티브 노드 그래프 (70%)    │  AI 해석 패널  │
│                                   │  (30%)       │
└───────────────────────────────────┴──────────────┘
```

**노드 타입 및 형태**:

| 노드 타입 | 형태 | 색상 | 예시 |
|----------|------|------|------|
| SWOT 요인 | 사각형 | 진한 회색 | W1(삼성 의존), O3(북미 고성장) |
| Weak Signal | 육각형 | 중간 회색 | 생성형AI, 리테일미디어 |
| Opportunity | 원형 | 검정 | AI 퍼포먼스 마케팅 |
| 클라이언트·산업 | 다이아몬드 | 연한 회색 | BYD, 모빌리티 |
| 지역 | 타원 | 흰색+테두리 | 북미, 유럽 |

**엣지 (연결선) 타입**:

| 관계 유형 | 선 스타일 | 설명 |
|----------|----------|------|
| 인과 강화 | 실선 굵음 | A가 B를 강화함 |
| 인과 약화 | 점선 | A가 B를 약화함 |
| 연결됨 | 실선 중간 | A와 B가 관련됨 |
| 공백 존재 | 점선 빨강 | A 영역에서 B가 부재함 |

- 엣지 굵기: 관계 강도(0~1) 비례
- 엣지 레이블: 관계 유형 텍스트 (마우스 오버 시 표시)

**지역 버튼 동작**:
- 지역 선택 시 해당 지역 무관 노드는 opacity 낮춤 (0.15)
- 해당 지역 노드·엣지만 선명하게 유지
- 전환 시 fade 애니메이션 적용
- `[전체]` 버튼으로 모든 노드 복원

**노드 클릭 인터랙션**:
1. 클릭된 노드와 1~2단계 거리 내 연결 path 하이라이트
2. 나머지 노드·엣지 opacity 낮춤
3. 하이라이트된 엣지 위에 관계 레이블 표시
4. 우측 AI 해석 패널: 해당 노드 중심의 인사이트로 교체
   - "이 노드는 [X]와 연결되며 [Y] 전략적 의미를 가집니다"

**Shift+클릭 (두 노드 동시 선택)**:
- 두 노드 사이의 최단 경로 하이라이트

### 6.6 Intelligence Feed 탭

**목적**: Weak Signal·Opportunity·SWOT 분석의 원천 데이터를 피드 형태로 제공

**레이아웃**:
```
[필터: 전체 | 뉴스 | 소셜 | 국내 | 북미 | 유럽]
┌─────────────────────────────────────────────┐
│  피드 아이템 리스트 (무한 스크롤)               │
│  각 아이템: 제목 | 출처 | 날짜 | 지역 배지 |     │
│            태그(연결된 신호·기회) | 북마크     │
└─────────────────────────────────────────────┘
```

**피드 아이템 구성**:
- 제목 (클릭 시 원문 링크)
- 출처 매체명 + 날짜
- 지역 배지 (국내 / 북미 / 유럽)
- 데이터 유형 배지 (뉴스 / 소셜)
- 연결 태그: 해당 아이템이 기여하는 신호·기회 항목
  - 예: `→ [t] 생성형AI 광고` `→ AI 퍼포먼스 마케팅`
- 북마크 아이콘

**필터 기능**:
- 데이터 유형 (뉴스 / 소셜)
- 지역 (국내 / 북미 / 유럽)
- 연결 태그 (특정 신호·기회 선택)

## 7. 추가 기능 명세

### 7.1 트렌드 타임라인

- Weak Signal 탭 드릴다운 패널 내 포함
- X축: 날짜 (최근 6개월)
- Y축: 등장 빈도 또는 기회점수
- 특정 신호가 weak → strong으로 진화하는 추이 선 그래프
- "이 신호는 N개월 전 처음 감지되었습니다" 텍스트 표시

### 7.2 신호 강도 게이지

- Overview 탭의 Weak Signal TOP 3 카드에 포함
- 0~100 수치의 수평 게이지 바
- 지난 주 대비 변화량: `▲ +12` 또는 `▼ -5` 형식
- 색상: 강도 높을수록 진한 색

## 9. 디자인 가이드

### 9.1 브랜드 컬러 제약

제일기획 브랜드 아이덴티티인 **블랙 & 화이트** 기반으로 디자인한다.

| 용도 | 색상 |
|------|------|
| 배경 (메인) | `#FFFFFF` |
| 배경 (서브) | `#F5F5F5` |
| 텍스트 (주) | `#000000` |
| 텍스트 (보조) | `#666666` |
| 텍스트 (힌트) | `#999999` |
| 테두리 | `#E0E0E0` |
| 강조 (버튼 활성) | `#000000` (검정 배경 + 흰 텍스트) |
| 차트 Weak Signal 점 | `#0A0A0A` |
| 차트 기타 신호 점 | `#A0A0A0` |
| Opportunity Underserved | `#E67820` (주황, 예외적 컬러 사용) |
| 지역 구분 배지 | 국내 `#333`, 북미 `#555`, 유럽 `#777` |

> **예외**: 차트의 Underserved 버블, 지역 배지 등 **데이터 구분이 필수적인 경우에만** 컬러 사용 허용. 그 외 모든 UI 요소는 블랙 & 화이트로 구성.

### 9.2 레이아웃 원칙

- 카드 컴포넌트: 흰 배경 + 0.5px 테두리 + border-radius 8px
- 차트 영역과 AI 해석 패널: 7:3 또는 6:4 그리드 분할
- 최소 마진: 컴포넌트 간 16px
- 탭 컨텐츠 패딩: 24px
